# 飞书 IM 集成 Spec

## REQ-0: 架构设计

**目标：** 在 OMP 中内嵌飞书 Bot 模块，支持多实例独立运行。

**约束：**
- 单个用户，多个 omp 实例，每个实例独立飞书对话
- 每个实例有独立的飞书 Bot App ID/Secret
- 费用按实例独立计算
- 危险操作需飞书内确认

**技术选型：**
- 飞书 Bot API v2（App ID + Secret + Event Subscription）
- WebSocket 长连接接收消息（或 HTTP 回调 + ngrok/frp）
- 复用现有 RPC Mode（`modes/rpc/`）
- 新建 `packages/coding-agent/src/im/` 模块

## REQ-1: 飞书 Bot 基础连接

**目标：** 实现飞书 Bot 基础连接，能接收和发送消息。

**验收标准：**
1. 支持 App ID + App Secret 获取 access_token
2. 支持 Event Subscription 接收消息（HTTP 回调或 WebSocket）
3. 支持发送文本/Markdown/Interactive 卡片
4. 支持消息签名验证（安全）

**测试：**
- [ ] 单元测试：token 获取逻辑
- [ ] 单元测试：消息签名验证
- [ ] 集成测试：发送测试消息到飞书

## REQ-2: 命令路由器

**目标：** 将飞书消息映射到 OMP RPC 命令。

**验收标准：**
1. 解析 `/omp <command>` 格式
2. 支持所有现有 OMP 命令（prompt/steer/follow_up/abort/branch 等）
3. 支持自然语言解析（可选，Phase 2）
4. 支持命令别名（/status → /omp get_state）

**测试：**
- [ ] 单元测试：命令解析
- [ ] 单元测试：命令映射表
- [ ] 集成测试：发送命令，验证 OMP 响应

## REQ-3: 状态同步

**目标：** 定期拉取 OMP 状态，推送到飞书。

**验收标准：**
1. 拉取 git 分支
2. 拉取上下文窗口消耗（tokens 使用量）
3. 拉取费用（cost）
4. 拉取当前任务（tool_execution 事件）
5. 支持手动触发状态查询（/omp status）
6. 支持定时推送（可选，Phase 2）

**测试：**
- [ ] 单元测试：状态数据提取
- [ ] 集成测试：拉取状态，验证数据准确性

## REQ-4: 输出格式化器

**目标：** 将 OMP Component/TUI 输出转换为飞书卡片格式。

**验收标准：**
1. 实现完整的 Markdown → 飞书卡片转换器
2. 支持标题/列表/代码块/表格
3. 支持语法高亮（代码块）
4. 支持流式输出（单条卡片持续刷新）
5. 支持错误信息格式化

**测试：**
- [ ] 单元测试：Markdown 解析
- [ ] 单元测试：飞书卡片生成
- [ ] 集成测试：OMP 输出 → 飞书卡片，验证渲染

## REQ-5: 确认机制

**目标：** 危险操作前发送确认卡片，用户点击后执行。

**验收标准：**
1. 识别危险操作（git push/commit/delete 等）
2. 发送 Interactive 卡片，包含确认/取消按钮
3. 等待用户点击，超时自动取消
4. 确认后执行命令，取消则中止
5. 支持预批准列表（某些命令无需确认）

**测试：**
- [ ] 单元测试：危险操作识别
- [ ] 单元测试：确认卡片生成
- [ ] 集成测试：发送危险命令，验证确认流程

## REQ-6: 多会话管理

**目标：** 支持多个 omp 实例独立运行，互不干扰。

**验收标准：**
1. 每个实例有独立的配置文件（飞书 App ID/Secret）
2. 每个实例有独立的 session 存储
3. 每个实例有独立的费用追踪
4. 支持配置热加载（无需重启）

**测试：**
- [ ] 单元测试：配置加载
- [ ] 集成测试：同时运行 2 个实例，验证隔离

## REQ-7: 插件入口

**目标：** 将 IM 模块注册到 OMP 生命周期。

**验收标准：**
1. 提供 `loadImPlugin()` 函数
2. 在 CLI 启动时可选加载（`--im` 标志）
3. 提供 `disableImPlugin()` 函数
4. 插件生命周期与 OMP 同步（启动/停止/重启）

**测试：**
- [ ] 单元测试：插件加载
- [ ] 集成测试：启动 OMP + 插件，验证正常工作

## 非功能性需求

**性能：**
- 消息响应延迟 < 1s（飞书推送）
- 流式输出刷新频率 < 500ms

**安全：**
- 所有 API 调用使用 HTTPS
- 消息签名验证
- 敏感信息脱敏（API Key/Password 等）

**可扩展性：**
- 插件架构，易于添加其他 IM（企业微信/钉钉/Telegram）
- 配置驱动，无需修改代码即可调整行为
